- Define the root `docker-compose.yml` to orchestrate all essential backing services and local environments.
- **Service 1: PostgreSQL (Relational Data Persistence)**
  - Use the image `postgres:15-alpine` to maintain a lightweight footprint.
  - Map explicit ports `5432:5432` to allow local host GUI management connections.
  - Define exact environment variables mapping to `.env` parameters explicitly: `POSTGRES_USER=gradion_user`, `POSTGRES_PASSWORD=secure_pass`, `POSTGRES_DB=expense_system`.
  - Mount a dedicated local Docker volume `./docker_data/postgres:/var/lib/postgresql/data` ensuring data persistence survives container teardowns and rebuilds during the development lifecycle.
- **Service 2: MinIO (S3 Mock Object Storage)**
  - Use the image `minio/minio:latest`.
  - Expose API ports `9000:9000` and the web console on `9001:9001`.
  - Map the access and secret keys via environment configurations `MINIO_ROOT_USER` and `MINIO_ROOT_PASSWORD` corresponding exactly to what the NestJS backend expects.
  - Execute the explicit command `server /data --console-address ":9001"` dynamically upon boot.
  - Automation Task: Introduce an adjacent orchestration container (`minio-setup`) running the `minio/mc` utility. This container will sleep until MinIO is responsive, subsequently execute `mc mb myminio/receipts`, configure a public read policy modifying `mc anonymous set download myminio/receipts`, and successfully exit. This prevents any manual bucket configurations during reviewer evaluations.
